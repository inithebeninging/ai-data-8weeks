# SQL Day 1

## 오늘 배운 것
- SELECT
- WHERE
- ORDER BY
- LIMIT

## 사용한 테이블
- sales

## 목표
판매 데이터를 사용해 원하는 컬럼 선택, 조건 필터링, 정렬, 상위 N개 조회를 연습한다.

##query1
SELECT *
FROM sales;
sales 테이블의 모든 데이터를 조회했다.

##query2
SELECT product
FROM sales;
Sales table 에서 products만 조회

##query3
SELECT region, product
FROM sales;
Sales 테이블에서 지역과 상품명 조회

##query4
SELECT *
FROM sales
WHERE region = 'Seoul';
서울 지역 데이터만 조회

##query5
SELECT *
FROM sales
WHERE quantity >= 5;
양이 5이상 데이터 조회

##query6
SELECT *
FROM sales
WHERE region = 'Seoul'
  AND category = 'Laptop';
서울지역의 노트북 판매 데이터만 조회

##query7
SELECT *
FROM sales
WHERE region = 'Seoul'
   OR region = 'Busan';
서울과 부산 지역 데이터 조회

##query8
SELECT *
FROM sales
WHERE category != 'Accessory';
악세서리가 아닌 데이터만 조회

##query9
SELECT *
FROM sales
ORDER BY quantity DESC;
전체 데이터를 양이 많은 순서로 정렬
키보드와 마우스등 컴퓨터 주변상품의 주문이 가장 많았다.

##query10
SELECT 
    product,
    quantity,
    unit_price,
    quantity * unit_price AS revenue
FROM sales;
판매금액 계산하기, as revenue 로 계산 결과 저장

##query11
SELECT 
    product,
    region,
    quantity,
    unit_price,
    quantity * unit_price AS revenue
FROM sales
ORDER BY revenue DESC
LIMIT 5;
판매금액 상위 5개 조회
값이 싼 노트북의 판매량이 많았다.


# 오늘 정리

오늘은 SELECT, WHERE, ORDER BY, LIMIT을 사용해서 원하는 데이터를 조회하는 방법을 연습했다.  
SELECT는 보고 싶은 컬럼을 고르고, WHERE는 조건에 맞는 행을 필터링하며, ORDER BY는 정렬, LIMIT은 조회 개수를 제한할 때 사용한다.
